问题1

求方程$x^2+(\alpha + \beta)x + 10^9 = 0$的根，其中$\alpha = -10^9, \beta=-1$，讨论如何设计计算格式才能有效减少误差，提高计算精度。

算法及分析

注意到，此处根$\alpha,\beta$数量级相差巨大，造成方程$x^2+bx+c=0$中$|b|$，$|c|$相差不大。于是在求根公式中计算$b^2-4c = (\alpha + \beta)^2 - 4\alpha\beta = (\beta - \alpha)^2$易近似为$\alpha^2$，造成显著计算误差。因此可以直接计算较大的根$\alpha$，并采用分母有理化求绝对值较小的根$\beta$，即

在程序编写中，注意到幂运算速度慢且结果不准确，我们采用b*b代替$b^2$即pow(b,2)。

### 分析

可以看出，算法二、三会稳定收敛到正确的结果，但算法二的收敛速度明显慢于算法三。

#### 算法一

$x_{k+1} = 15-2x_k^2$可以转化为$x_{k+1} - x^\star = -2(x_k-x^\star)^2-10(x_k-x^\star)$。于是
$$\frac{x_{n+1} - x^\star}{x_{n}-x^\star} = -2(x_{n}-x^\star) - 10$$

若$\{x_n\}$收敛于$x^\star$，则
$$\left|\lim_{n \to \infty}\frac{x_{n+1} - x^\star}{x_{n}-x^\star}\right| = |-10| = 10 > 1$$
矛盾！说明$\{x_n\}$不收敛于$x^\star$。

#### 算法二

注意到，$f(x) = \frac{15}{2x+1}$的不动点$x^\star=2.5$是函数的不动点。

易于得到，$x_1 = 3,x_2 = 15/7$，则根据归纳法易于证明
$$\frac{15}{7} \leq x_{k+1} \leq 3,k \geq 0$$
这等价于
$$\frac{15}{7} \leq \frac{15}{2x_k+1} \leq 3$$
即
$$2 \leq x_k \leq 3$$
于是$x_k\in [15/7,3], \forall k \geq 1$。

类似地，可以证明$x_k \in [555/457,3], \forall k \geq 3$,

于是，$\forall x,y \in [555/457,3]$，有
$$\left|\frac{f(y) - f(x)}{y-x}\right| = \frac{30}{(2x+1)(2y+1)} \leq \frac{30}{(2\times\frac{555}{457}+1)^2} < 1$$
于是根据**Banach不动点定理**，$f(x)$在区间$[555/457,3]$内不动点迭代收敛，即$\{x_k\}$收敛。

此时，
$$\lim_{n \to \infty}\frac{x_{n+1} - x^\star}{x_n - x^\star} = -\lim_{n \to \infty}\frac{5}{2x_n+1} = -\frac{5}{6} \in (-1,1)$$
而
$$\lim_{n \to \infty}\frac{x_{n+1} - x^\star}{(x_n-x^\star)^2} = -\lim_{n \to \infty}\frac{5}{6(x_n-x^\star)} = \infty$$
于是该方法是线性收敛的。

#### 算法三

算法三是采用**牛顿法**迭代的例子。

类似实验二，可以确定$\{x_k\}$所在的区间位于$[2,3]$内。

注意到$\forall x,y \in [2,3]$，
$$\frac{f(y)-f(x)}{y-x} = \frac{1}{2} - \frac{121}{2(4x+1)(4y+1)} < \frac{1}{2}$$
于是根据**Banach不动点定理**该方法收敛。

又，
$$\lim_{n \to \infty}\frac{x_{n+1} - x^\star}{(x_n - x^\star)^2} = \frac{x_n - \frac{2x_n^2+x_n-15}{4x_n+1} - x^\star}{(x_n - x^\star)^2} = \frac{4x^2-20x+25}{(2x-5)^2} = 1$$
于是该方法是二次收敛的。

于是算法三收敛速度明显快于算法二。

### 错位法

经过程序测试，发现错位法存在其中一个端点值永不被更新的缺点。实际上，只要函数在初值对应的区间$(x0,x1)$内二阶导不变号(即**凸性不变**)，就一定会产生这样的情况。根据集合意义容易看出该结论的正确性。具体程序如下：

```cpp 测试程序2.4.4_错位法
#include <iostream>
#include <cmath>

using namespace std;

double f(double x){
    return 5.0*x - exp(x);    
}

int sgn(double x){
    int sgnx = (x>0) ? 1 : -1;
    return sgnx;
}

int main(){
    double x0 = 0;
    double x1 = 1;
    double x;
    int N = 0; //iterations
    while (ceil(x0*10000) != ceil(x1*10000) && N<10000) {
        x = x1 - f(x1)*(x1-x0)/(f(x1)-f(x0));
        if (sgn(x0) == sgn(x)) x0 = x;
        else x1 = x;
        N++;
//        cout<<x0<<" "<<x1<<endl;
    }

    printf("x=%.4f\nN=%d",x,N);
    return 0;
}
