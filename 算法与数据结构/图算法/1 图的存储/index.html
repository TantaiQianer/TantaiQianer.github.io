这是一篇关于图的存储方式的介绍。

# 概述

图可以分成稠密图和稀疏图。一般边数与点数之间为线性关系为稀疏图，近似为二次关系则为稠密图。此外，根据关系是否是对称的，采用有向图或无向图进行表示。对于路径、圈、树等基本概念，在此不再重复。

在下面的算法表述和复杂度分析中，我们默认采用 VVV 表示顶点集，采用 EEE 表示边集，采用 ∣V∣|V|∣V∣ 或 nnn 表示顶点数，采用 ∣E∣|E|∣E∣ 或 mmm 表示边数。

# 图的存储

由于图的规模不同，图中顶点和边的关系也不同，于是对于图，我们需要采用不同的方式进行图的存储。

# 关联矩阵

关联矩阵即采用一个大小为 n×mn \times mn×m 的二维数组存储一个图。其中，每一行代表一个顶点，每一列代表一条边，如果某一条边eje_jej​ 的端点为某一个顶点 viv_ivi​, 那么我们将 (i,j)(i,j)(i,j) 处的元素为 111, 否则为 000. 对于有向图，我们一般规定贡献出度的边标 111, 贡献入度的边标 −1-1−1.

关联矩阵的优点在于比较清晰，缺点在于矩阵中的大多数元素被浪费了。且图较稠密时，矩阵更加庞大。

# 邻接矩阵

邻接矩阵是处理较小规模的图时常常采用的存储方法。邻接矩阵的大小为 n×nn \times nn×n, 其中若点 viv_ivi​ 与点 vjv_jvj​ 之间有边相连，那么 (i,j)(i,j)(i,j) 处及 (j,i)(j,i)(j,i) 处的元素为 111, 否则为 000. 类似地，对于有向图，我们同样规定贡献出度的边标 111, 贡献入度的边标 −1-1−1.

在此规定下，对一个邻接矩阵 XXX, 若 Xi,jk=lX_{i,j}^k=lXi,jk​=l, 则说明从 viv_ivi​ 至 vjv_jvj​ 的长度为 kkk 的有向路径有 lll 条。

如果一个图是加权的，那么我们可以将 (i,j)(i,j)(i,j) 处的值改为边 vivjv_iv_jvi​vj​ 的权重。这样上述有向路径的结论便不再成立。

# 复杂度

  • 查询是否存在某条边：O(1)O(1)O(1)。
  • 遍历一个点的所有出边：O(n)O(n)O(n)。
  • 遍历整张图：O(n2)O(n^2)O(n2)。
  • 空间复杂度：O(n2)O(n^2)O(n2)。

应用

邻接矩阵仍然存在空间浪费的问题。如果图较稀疏，那么图中的相当一部分空间都是被浪费的。于是，我们在此基础上优化，得到了邻接表方法。

# 邻接表

邻接表是一个列表数组。首先，我们建立一个长度为nnn 的数组 a ，其中的每一项是一个链表 List 。链表与图中的顶点一一对应，链表中的每一项存储该顶点可以到达的顶点。因此，采用邻接表存储可以有效解决从一个点进行遍历的问题，同时对于稀疏矩阵更节约空间。但是缺点在于链表的访问速度慢于数组。

对于稀疏图GGG, 我们可以采用十字链表存储其邻接矩阵。十字链表是一种特殊的数据结构，每一个节点含有两个指针域，分别指向同行和同列的下一个节点。这样，对于稀疏矩阵，我们在节约空间的情况下，以O(d(v))O(d(v))O(d(v)) 复杂度访问一个节点指向和被指向的全部节点，这是较快的。

# 并查集算法

对于无向图，可以采用并查集进行判断。但并查集的缺点在于不能将两个合并的集合分离，即不能处理删除边的情况。于是只能用在不断添加边的过程中。

# 最大流问题

在有向图中，每个边eije_{ij}eij​ 权非负，边权cijc_{ij}cij​ 称为容量即额定最大流量。我们需为每个边eije_{ij}eij​ 另赋一个权f(eij)f(e_{ij})f(eij​), 称为边的流量，其中流量f(eij)f(e_{ij})f(eij​) 应小于容量cijc_{ij}cij​.

网络图中的顶点分为三类， S 称为源， V 称为汇，各仅有一个。其余节点没有存储能力，即流入边权和等于流出边权和，可以表示为∑e.in.vf(e)=∑e.out.vf(e)\sum_{e.in.v}f(e) = \sum_{e.out.v}f(e)∑e.in.v​f(e)=∑e.out.v​f(e).

我们称一个图的总流量为流出源点 (或流入汇点) 的所有流量之和，即∑e.out.sf(e)=∑e.in.vf(e)\sum_{e.out.s}f(e) = \sum_{e.in.v}f(e)∑e.out.s​f(e)=∑e.in.v​f(e).

我们所希望寻找的是从源流出的流量最大的流 (或流入汇的流量最大的流)，称为最大流问题。

# Ford-Fulkerson 方法

Ford-Fulkerson 方法即标号方法，又称增广路方法，是 20 世纪 50 年代福特 (Ford) 和福克逊 (Fulkerson) 提出的方法。之所以称之为方法而不是算法，是因为其仅给出了最基本的想法，但是无法确定最终找到的方式。实际上，下面几种最大流算法几乎都是基于此方法的。

首先，我们引入算法中的一些基本概念。

# 可行路径与增广路径

对一个网络流GGG, 我们对一条从源点 S 到汇点 T 的路径赋流量fff, 路径上每条边eije_{ij}eij​ 都满足f<cijf<c_{ij}f<cij​, 那么我们称这个路径为一个可行路径。

# 残量网络 (residual network)

该算法的想法是，从某个可行流开始，构建一个残差网络，在网络中寻找可能的增广路径，然后将可能的增广路径添加到原可行流中。当无法找到增广路径时，就得到了最大流。
