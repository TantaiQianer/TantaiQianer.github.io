# B 树

B 树 (B-tree) 是一种自平衡树，于 1970 年被提出 (论文地址，一个排版较好的版本)。

# 定义和基本性质

关于 B 树的术语在文献中没能实现很好的统一。例如对 B 树阶 (order) 的定义不明确。下面，我们选取两种基本的定义方式，以了解 B 树的设计思想。

# Knuth 的定义

Knuth 关于 B 树的定义如下：

mmm 阶 B 树是满足如下特性的树：

  1. 每个节点最多有 mmm 个子节点。
  2. 每一个非叶子节点（除根节点）最少有 ⌈m2⌉\lceil \dfrac{m}{2} \rceil⌈2m​⌉ 个子节点。
  3. 非叶子节点至少有 222 个子节点。
  4. 有 kkk 个子节点的非叶子节点拥有 k−1k−1k−1 个键。
  5. 所有的叶子节点都在同一层。

# 《算法导论》中的定义

在《算法导论》中也有对 B 树的定义。其定义如下：

一个 B 树 TTT 是一个根为 T.root 的有根树 (rooted tree). 其有如下性质：

  • 每一个节点 xxx 满足如下性质：
    • x.nx.nx.n 是节点 xxx 当前存储的键 (key) 数目。
    • xxx 的 x.nx.nx.n 个键 x.key1,x.key2,…,x.keyx.nx.key_1, x.key_2, \dots, x.key_{x.n}x.key1​,x.key2​,…,x.keyx.n​ 按单调递增顺序存储，即 x.key1≤x.key2≤⋯≤x.keyx.nx.key_1\leq x.key_2 \leq \dots \leq x.key_{x.n}x.key1​≤x.key2​≤⋯≤x.keyx.n​.
    • x.leafx.leafx.leaf 是一个布尔值，表明 xxx 是否是一个叶子节点。
  • 对于每一个内部节点 (internal node, 即非叶子节点), 其拥有 x.n+1x.n+1x.n+1 个孩子 x.c1,x.c2,…,x.cx.n+1x.c_1, x.c_2, \dots, x.c_{x.n+1}x.c1​,x.c2​,…,x.cx.n+1​.
  • 叶子节点没有孩子，其 ccc 域未定义
  • 键 x.keyix.key_ix.keyi​ 将子树中存储的众多键进行了划分。即若 kik_iki​ 是存储在根为 x.cix.c_ix.ci​ 子树中的任一节点，那么

    k1≤x.key1≤k2≤⋯≤x.keyx.n≤kx.n+1k_1 \leq x.key_1 \leq k_2 \leq \cdots \leq x.key_{x.n} \leq k_{x.n+1}k1​≤x.key1​≤k2​≤⋯≤x.keyx.n​≤kx.n+1​

  • 所有的叶子节点有相同的深度，均为树高度 hhh.
  • 节点容纳的键数量存在上界和下界，这通过 B 树的最小度 (minimum degree) ttt 定义：
    • 每一个非根节点含有至少 t−1t-1t−1 个键。每一个内部节点含有至少 ttt 个孩子。如果该树非空，那么根节点至少拥有一个键。
    • 每一个节点含有至多 2t−12t-12t−1 个键。因此，每一个内部节点含有至多 2t2t2t 个孩子。如果一个节点含有 2t−12t-12t−1 个键，那么称这个节点是满 (full) 的。

最简单的 B 树的最小度 t=2t=2t=2. 这样，每个内部节点的孩子数目为 2,32, 32,3 或 444. 因此称其为一个 2-3-4 树。在实际应用中，ttt 值往往很大。
