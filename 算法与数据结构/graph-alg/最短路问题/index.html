需要注意的是，以 sss 点为源点跑 Bellman–Ford 算法时，如果没有给出存在负环的结果，只能说明从 sss 点出发不能抵达一个负环，而不能说明图上不存在负环。

因此如果需要判断整个图上是否存在负环，最严谨的做法是建立一个超级源点，向图上每个节点连一条权值为 000 的边，然后以超级源点为起点执行 Bellman–Ford 算法。

# Dijkstra 算法

是一种求解非负权图上单源最短路径的算法。

这是暴力的写法，复杂度是 O(n2+m)=O(n2)O(n^2 + m) = O(n^2)O(n2+m)=O(n2) 的。

如果对于稀疏图，采用二叉堆维护边的插入情况，那么复杂度可以优化到 O((n+m)log⁡n)O((n+m) \log n)O((n+m)logn). 对于稀疏图 (m<n2/log⁡nm < n^2/\log nm<n2/logn), 这是一个不错的优化。

如果采用斐波那契堆优化，那么复杂度可以降低到 O(nlog⁡n+m)O(n \log n+m)O(nlogn+m).

# Floyd-Warshall 算法

是否存在负权，只需要观察 f[n][i][j] 中的对角线元素 ( i==j ) 中是否存在负元即可。

# Johnson 算法（稀疏图上）

复杂度 O(n2log⁡n+nm)O(n^2 \log n + nm)O(n2logn+nm).

步骤如下：

  1. 添加超级源点 sss 并初始化边。(O(n)O(n)O(n))
  2. 运行 Bellman-Ford 算法，判断是否存在负环。(O(n2+nm)O(n^2+nm)O(n2+nm))
  3. 若不存在，根据 Bellman-Ford 计算的结果进行重赋权。(O(n+m)O(n+m)O(n+m))
  4. 对各节点，运行 Dijkstra. (O(n(nlog⁡n+m))=O(n2log⁡n+mn)O(n(n \log n+m)) = O(n^2 \log n + mn)O(n(nlogn+m))=O(n2logn+mn)).
