# 基本概念

操作系统主要着眼于三点，分别是虚拟化 (virtualization)、并发 (concurrency) 和持久性 (persistence)。

# 虚拟化

虚拟化的目的是使程序的运行更加容易。

操作系统将物理资源进行抽象，转化成更通用、更便捷的虚拟形式，就是操作系统进行虚拟化的过程。因此，操作系统有时也被称为虚拟机 (virtual machine)。(实际上，在计算机组成原理中我们曾了解，计算机系统中每一级都是下一级的抽象，因此每一级都可以在广义上视作下一级的虚拟机。) 为使用户和应用程序可以利用虚拟机，操作系统需要给出一些可供调用的应用程序编程接口 (application programming interface, API)。其中，用于调用操作系统功能的 API 称为系统调用 (system call)，是应用程序与系统间的接口。

此外，由于虚拟化过程中操作系统让许多程序可以同时访问相同的设备和物理存储空间，因此起到了资源配置和资源共享的作用。于是，操作系统又被称为资源管理器 (resource manager)。
